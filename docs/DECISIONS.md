## 2026-06-07 — 다운로드를 고정 트레일링 윈도우로 재설계

정체의 근원은 download_update가 전체 테이블의 글로벌 MAX(date)를 모든 종목 공통 start로 쓴 것이다(일부 종목이 앞서면 나머지가 영구히 굶음). 매 거래일 모든 종목에 "최근 N거래일(7~10일)"을 일괄 재요청·upsert하는 방식으로 바꾼다 — 부분 실패가 다음 실행에서 자동 치유되어 정체가 구조적으로 불가능해진다. 종목별 워터마크는 가장 정확하나 호출량이 폭증해 rate-limit 위험이 커서 배제. 대규모 과거 공백은 일회성 재구축으로 메운다.

## 2026-06-07 — 외부 의존성 정확 고정(==) + lock 파일 이중화

이번 사태의 근본 원인은 상한 없는 `>=` 핀이라 매일 CI가 최신을 끌어와 yfinance 0.2→1.4, pandas 2→3 메이저 점프로 깨진 것. 매일 무인으로 도는 운영 파이프라인은 재현성이 자동 패치 수신보다 압도적으로 중요하므로, CI가 설치하는 의존성만 `requirements.lock`으로 정확 고정한다. 단 PyPI로 배포되는 엔진의 pyproject 의존성까지 ==로 못박으면 사용자 환경과 충돌하므로, 배포용은 느슨하게 두는 이중화로 간다. 업그레이드는 사람이 PR에서 테스트 후 의도적으로만.

## 2026-06-07 — yfinance 실패 종목 감지를 반환 커버리지 검사로 교체

옛 `yf.shared._ERRORS` 전역 dict는 새 yfinance에서 호출별 컨텍스트에 담겼다 폐기되어 외부 공개 API가 없다(아키텍처적 제거라 같은 정보를 다른 이름으로 못 꺼냄). "요청한 티커 집합 − 데이터가 돌아온 티커 집합 = 실패"로 감지한다 — 라이브러리 내부에 의존하지 않아 향후 버전 변화에도 깨지지 않는다. 로그 파싱은 포맷 의존이라 또 다른 무방비를 낳으므로 배제.

## 2026-06-07 — 기준 지수(SPY/QQQ)는 모집단에 포함해 함께 랭크

config 주석은 "순위 제외"였으나 코드·실데이터는 "함께 랭크"로 모순 상태였다. PM 의도는 "일반 ETF는 제외하되 SPY/QQQ는 지수로서 전체 시장 퍼포먼스를 확인하는 용도"이며, 이는 지수에 RS Rating을 부여해야 가능하다. 따라서 코드 동작(함께 랭크)을 유지하고 거짓 주석만 수정한다. 4,600개 중 2개라 다른 종목 백분위 왜곡은 무시할 수준.

## 2026-06-07 — 재구축 범위: 2025-03-21~현재 전체 RS 재계산, 4/20~5/22 전량 폐기

신뢰 가능한 RS는 2025-03-21(워밍업 종료)~2026-04-17에만 존재하고 그 이전은 데이터 부족, 4/20 이후는 정체로 전량 깨졌다. 4/17 이전 RS도 동일 의심 코드의 산물이므로 신뢰 구간 전체를 한 번 깨끗이 재생성·검증한다(가격 close는 기존 재활용해 비용 최소화). 4/20~5/22의 54~75종목 RS는 "맞아 보이지만 틀린"(소수 모집단 백분위) 값이라 ON CONFLICT 덮어쓰기로 전량 재계산한다.

## 2026-06-07 — Retention: close 가격만 13개월 prune, RS 등급은 영구 보존

PRICE_RETENTION_MONTHS=13이 선언만 되고 미구현이라 데이터가 무한 증가하며 매일 pivot 계산이 느려져 타임아웃→정체를 악화한다. RS 계산 입력인 close만 13개월로 prune해 매일 계산량을 일정하게 유지하고, 결과인 rs_rating은 과거 등급 조회를 위해 영구 보존한다.

## 2026-06-07 — RS Rating 모집단: 불완전한 거래일은 산출하지 않음

정체 버그로 한 거래일의 유효 종목이 54개로 줄면 그 안의 백분위는 무의미해진다(54개 중 1등도 99). 모집단을 "충분한 히스토리를 가진 전체 유니버스"로 정의하고, 데이터 파이프라인이 매일 ~4,600종목을 채우도록 고치되, 방어선으로 유효 종목이 임계치(예: 90%) 미만인 날은 RS Rating을 저장하지 않는다. forward-fill(죽은 가격으로 만든 가짜 모멘텀)은 RS를 더 오염시키므로 배제.

## 2026-06-07 — 252거래일 미만 종목은 RS 없음

RS Raw는 252거래일(약 1년) 룩백이 필요하다. 이력이 짧은 신규 상장주에 짧은 데이터로 RS를 부여하면 "같은 척도 비교"가 깨진다(사과와 오렌지). IBD 정통대로 충분한 히스토리가 쌓일 때까지 RS를 보류한다. '매일 모든 종목'의 '모든'은 '충분한 히스토리를 가진 모든 종목'을 뜻한다. RS 공식 가중치(0.4/0.2/0.2/0.2 × 63/126/189/252거래일)는 널리 통용되는 IBD 근사로, 과거와의 연속성을 위해 유지.
